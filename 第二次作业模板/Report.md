## 1. 页面结构

页面主要分成了四块，结构比较清楚，也方便后面写交互。

- **header**：页面标题
- **timetable**：课程表主体（表格/网格），展示一周的课程安排
- **add-panel**：添加科目的表单
- **search-panel**：查询功能区（按科目查时间 / 按日期查教室）

## 2. 功能实现说明

| 功能 | 实现方式 |
|------|----------|
| 添加科目 | 通过表单提交课程名、老师、教室、星期和节次，生成课程对象后加入数组并重新渲染课程表 |
| 删除科目 | 在课程卡片右上角放一个 × 按钮，点击后从课程数组中删除对应课程，再刷新页面显示 |
| 拖拽科目 | 给课程卡片设置 `draggable="true"`，在 `dragstart` 里记录课程 id，在目标格子 `drop` 时更新课程的星期和节次 |
| 查询科目时间 | 输入科目名称后，在课程数组里做模糊匹配，筛选出对应课程并展示时间信息 |
| 查询教室行程 | 选择星期后筛选当天课程，按节次排序后展示出来，方便查看当天安排 |
| 数据持久化 | 使用 `localStorage.setItem('courses', JSON.stringify(courses))` 保存数据，刷新后再通过 `JSON.parse(localStorage.getItem('courses'))` 恢复 |

## 3. 数据结构设计

采用的是一个课程对象数组，所有课程都存在 `courses` 这个数组里。每一门课的字段比较简单，够用即可。

```js
[
  {
    id: "唯一标识",
    name: "高等数学",
    teacher: "张老师",
    room: "A201",
    day: 1,
    period: 1
  }
]
```

## 4. 拖拽功能实现

拖拽部分主要按 HTML5 的原生 Drag and Drop API 来实现，整体思路不复杂，但细节需要处理到位。

- **dragstart**：在课程卡片开始拖动的时候，把课程 `id` 存到 `dataTransfer` 里，这样后面放到别的格子时可以找到这门课。
- **dragover**：在时间单元格里调用 `event.preventDefault()`，不然浏览器默认不允许放置。
- **drop**：读取拖拽出来的课程 `id`，找到对应课程对象，再把它的 `day` 和 `period` 改成目标格子的值，最后重新保存并渲染。

## 5. LocalStorage 使用

数据持久化的实现方式比较直接，就是把课程数组序列化后放进 `localStorage`。

- 保存时机：添加课程、删除课程、拖拽课程之后都会保存一次，避免刷新后丢数据。
- 恢复时机：页面刚加载时先读取 `localStorage`，如果里面有数据就直接恢复到 `courses` 数组里。
- 存储格式：字符串格式，具体是 `JSON.stringify(courses)` 存进去，读取时再 `JSON.parse(...)` 转回来。

## 6. 遇到的问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 刚开始拖拽后 drop 不生效 | 后面发现是漏写了 `dragover` 里的 `event.preventDefault()`，补上以后就正常了 |
| 课程卡片重复渲染 | 每次重新渲染前先把课程格子清空，再统一根据数组重绘 |
| 删除按钮点击后会影响拖拽 | 在删除按钮点击事件里加了 `event.stopPropagation()`，避免事件冒泡干扰卡片本身 |
| 查询结果不好看 | 单独做了结果区域的样式，让查询结果能分块显示，读起来更清楚 |
